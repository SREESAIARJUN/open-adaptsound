Why this exists

Windows plays one generic mix. It does not know your ears, your headphones, or which bands you hear less well. Music-player EQs stay in one app. Manual Equalizer APO is powerful but easy to mis-set. Open-AdaptSound measures how you hear and writes a real system-wide profile.

Without it

  • Factory sound for a “generic” listener
  • EQ only inside Spotify / one player
  • Guessing which bands to boost
  • Same curve for both ears

With Open-AdaptSound

  • Matched to how you hear
  • System-wide via Equalizer APO
  • Beep Yes/No finds thresholds
  • Independent left / right peaks

How it works

  1. 1 Calibrated hearing check System volume locks to a fixed reference (auto-restored after). Pure sine beeps, left then right, 15 bands to 16 kHz. Answer Yes or No — a decibel-domain staircase finds your true threshold per band.
  2. 2 Profile Thresholds become dB gains relative to your best band + preamp headroom (Gentle / Balanced / Full).
  3. 3 Apply Writes Equalizer APO config (hot-reload, keeps your other filters).
  4. 4 Use Every app is equalized at the Windows stack. This app leaves the path.

headphones → beep test → Open-AdaptSound → Equalizer APO → all apps

FAQ

Is this a medical hearing test?

No. Practical headphone check for EQ—not a clinical audiogram.

Do I need Equalizer APO?

Yes for system-wide results. This app only writes the profile.

Will it add game latency?

After Apply, EQ runs in Equalizer APO (low latency). Open-AdaptSound is not processing live audio.

Headphones or speakers?

Designed for headphones so left/right tones stay isolated.

Why does my volume change during the test?

The test locks Windows volume to a fixed reference level so results don't depend on where your slider happens to be — the same trick phone hearing tests use. Your previous volume is restored automatically.

Does it wipe my existing APO config?

No. It adds an Include file for the personal profile.
